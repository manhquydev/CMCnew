# Phase 2: AGENTS/CLAUDE hard rule

## Overview

Add a mandatory (not advisory) pre-edit lookup rule to `AGENTS.md` and `CLAUDE.md` — the two always-loaded context files — so an agent MUST consult `docs/DECISION_INDEX.md` + the linked decision doc before editing a governed file, and restate the rule in its response before coding.

## Requirements

- Functional: rule text added to both files; references the index file Phase 1 created; points to the EXISTING `.claude/rules/review-audit-self-decision.md` "User Decisions" section for the escalation path (do not duplicate that content, just link it).
- Non-functional: **zero disruption to existing structure** — both files have working `@import`-style references (`CLAUDE.md` has bare `@AGENTS.md` / `@docs/FEATURE_INTAKE.md` lines under `## Harness`; `AGENTS.md` has its own `## Branch workflow` / `## Harness` / GitNexus sections). Do not touch the `@` import lines. Do not reformat unrelated sections.

**Exact section text to insert (both files, identical):**

```markdown
## Decision Lookup (Hard Rule)

Before editing any file matched by a row in `docs/DECISION_INDEX.md`, MUST:
1. Read the linked decision doc under `docs/decisions/`.
2. Restate the governing rule in the response, before writing any code.
3. If the planned change would contradict that rule, STOP — do not silently
   reinterpret or override it. Follow `.claude/rules/review-audit-self-decision.md`
   → "User Decisions": present the original decision, the new concern, the
   trade-off, and concrete options, then wait for the user.

Only update `docs/DECISION_INDEX.md` when: (a) a new decision doc is created per
`docs/FEATURE_INTAKE.md`'s hard-gate, or (b) the user explicitly confirms a
changed decision mid-work (new decision doc supersedes the old one; index row
updated to point at it; old doc kept, not deleted). Never add or edit an index
row speculatively without a backing decision doc.
```

# Phase 6: Parent student validation

## Overview

Prove the full requested loop from director setup to teacher action to parent/student LMS experience, while reusing existing tested flows where possible. This is smoke/fix validation, not a redesign phase.

Likely touchpoints:

- `apps/lms/src/App.tsx`
- `apps/lms/src/parent-view.tsx`
- `apps/lms/src/student-view.tsx`
- `packages/ui/src/lms-login-gate.tsx`
- `apps/api/src/routers/lms-auth.ts`
- `packages/auth/src/lms.ts`
- `apps/api/src/routers/submission.ts`
- `apps/api/src/routers/session-evidence.ts`

## Implementation Steps

1. Build a deterministic test fixture or seed flow:
   - Director user.
   - Teacher user.
   - Parent phone/email.
   - Student.
   - Class with schedule and teacher assignment.
   - Curriculum unit and exercise.
2. Run the full role path with seeded/API fixtures where live MFA/email blocks automation:
   - Director creates or provisions family/student per accepted decision.
   - Director creates class/adds student/upload exercise.
   - Teacher marks attendance/publishes evidence/grades submission.
   - Parent logs in and sees evidence/grade.
   - Student logs in using family phone/profile picker and submits homework.
3. Validate security:
   - Parent cannot see another family's student.
   - Student cannot access another student's submission/evidence.
   - Teacher cannot mutate unassigned/cross-teacher/cross-facility data unless server permissions explicitly permit.
4. Validate email:
   - Parent welcome/notification email outbox queues with `ParentAccount.email`.
   - External parent mail drains through the configured Brevo path in the launch environment.
   - A non-secret test recipient reaches `sent` status or the deployment is blocked with a clear operator action.
   - Brevo/Graph split remains as decision 0030 describes.
5. Validate privacy:
   - Reports and audit rows generated by new code do not include raw parent phone/email.
6. Capture screenshots/artifacts for the guide only after tests pass.

## Success Criteria

- [x] Full loop passes locally or on dev with deterministic data.
- [x] Parent view shows teacher-published comments/photos and grade state.
- [x] Student can submit homework and retrieve their own layer.
- [x] Unauthorized family/student access attempts fail.
- [x] Email outbox has expected parent notification rows.
  - Production outbox check on 2026-07-06 shows `brevo|sent|4` with `failed=0` and `queued=0`.
- [x] At least one parent-facing test email reaches sent/delivered-equivalent state in the target environment, or Brevo readiness is listed as a blocker.
  - Production `.env.production` has Brevo API key, sender email, and sender name set; no secret values were printed.
- [x] Evidence is recorded in Harness trace.
- [x] New reports/audit rows avoid raw parent phone/email.
  - No new parent PII-bearing report/audit path was added.

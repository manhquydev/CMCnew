### Attacked and SURVIVED (documented non-issues)

- **TZ formula (P2)**: `Date.UTC(y,m,d,hh-7,mm)` from sessionDate's UTC components is correct — `Date.UTC` normalizes negative/overflow hours, so a 01:00 ICT end rolls to the prior UTC day correctly and 23:30 ICT = 16:30 UTC same day. Session writers store date-string-parsed UTC-midnight dates (`schedule.ts:176,187`), consistent with the read side. The design holds; keep the pinned-TZ boundary tests.
- **Submission read isolation (P1 req 4)**: `submission_isolation` already exists independent of Exercise (`20260623100000_principal_aware_rls/migration.sql:53-61`, facility for staff + `student_id = ANY(app_student_ids())` for principals). Read-side is safe; only the WRITE path (C2) leaks.
- **`gradeFromPercent` keep-decision (D5)**: confirmed pure (`packages/domain-grading/src/grading.ts:130-133`), no DB read of thresholds; only seed-demo `:83`, parity test `:104`, and the schema relation `:877` touch GradingThreshold — matches P1's corrected inventory.
- **Grade/star flows**: `grade.ts:97-108` reads `starReward` via the submission→exercise relation — unaffected by column drops.
- **Validation Summary teacher-read item**: consistent with P2 req 4 and `grading.tsx:24,515` — no contradiction (the contradiction is the due/dueOffset residue, M3).
